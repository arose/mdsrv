Viewing simulations

To view simulations provided by the MDsrv, only a link to the session (e.g. provided by a collaborator) and on demand a user and a password is needed. The NGL Viewer requires your browser to support WebGL which is standard in almost all modern web browser. Therefore, there is no need to install anything. If you run into problems, please consult the FAQ.

Quick usage guide

Please open the provided URL (e.g. http://proteinformatics.charite.de/MDsrv-example2) and enter the user & passwort if necessarry.

In the following we will refer to the standard NGL GUI symbols which are explained in the figure above. NOTE: Click on the figure to increase the size.


Play the simulation by pressing the button (A). The frames (B) and if the delta-time was provided also the runtime (C) is shown.


The trajectory can be adjusted within the trajectory menu (reachable by clicking on the menu symbol next to (C) from the previous picture). There, centering, removal of periodic boundary condition (PBC) and superpositioning can be done with regards to the selection stated in the trajectory funnel (here "backbone and not hydrogen", located above the play button (A) in the previous figure).
Other options are: step size, interpolation type (none, spline, linear), interpolation steps, play timeout, play direction (forward, backward), play mode (loop, once), delta time and time offset.


Structures are load with a default representation or as prepared by the expert. Still, the GUI provides the possibility to add new representations for each structure by entering the structure menu and add a new representation.


Each representation can be changed and modified by entering the menu located next to the representation (over its filter field). On the right, options for modifying the cartoon representation are shown.


NGL selection language

The content of a structure can be specified through the filter selection input field (e.g. "/0 and not hydrogen"). The same applies for each representation. Note that if a selection is specified via the structure filter selection (parent filter field), those also apply for the simulation and representation filters (does not need to be mentioned again).

  • * everythin/all.
  • not _h no hydrogens
  • protein protein (no ions, lipids, water, HETATMs)
  • hetero hetero atoms / HETATM
  • :B chain
  • .CA atom
  • @703 atomindex
  • 10-15 range of residues
  • /0 number of model within the structure file

Further information

The NGL GUI provides also other options:
Within the File menu (A), new structures or other files can be loaded (local ones via Open..., deployed ones via Import..., PDBs via entering the PDB-ID) also with some preferences and images can be generated (simple via Screenshot and with further options via Export image...). In general, new structures can also be loaded via drag'n'drop.
Within the View menu (B), the gui settings can be changed. The Help menu (C) provides more information and can also change the general Preferences.
A number of clickable icons provide common actions. Most icons can be clicked on, just try it or hover the mouse pointer over it to see a tooltip.

Loading simulations

Simulations can be loaded in several different ways into the NGL GUI. The main focus of the MDsrv is to make deposited (remote) simulations available to others. Still, we provide an option for viewers (meaning those who have received a MDsrv session) to load their local trajectories, e.g. for comparison with the deposited simulations data into the session. Go here to find more about how to load a trajectory.
