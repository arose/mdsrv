MDsrv serving

Local usage

Within a local environment, for short-time usage or testing, the MDsrv can be started from a directory providing access to files within the child directories by one of the following commands:

mdsrv
mdsrv struc.gro traj.xtc
mdsrv --cfg my_conf.cfg

The first command starts the MDsrv including the NGL viewer with access to files within a subdirectories, exclusively. The second command loads a structure file (e.g. a .gro or .pdb file) and if provided also a trajectory into the MD viewer (a structure file is mandatory for loading a trajectory). The underlying trajectories can still be loaded. For more intense usage, the configuration file (.cfg, explained below) can be loaded by using the --cfg flag of the third command.

System wide/web-server usage

To maintain MDsrv as a system wide web-server, follow the installation instructions. Besides apache, any webserver with wsgi support can be used (not tested).

We provide an example of the usage of the MDsrv as a web-server in combination with the NGL viewer. By utilizing a web-server, collaborators can easily inspect the simulations visually on-the-fly.

Loading trajectories

To load simulations or .ngl scripts from an underlying directory into the NGL viewer, browse through the folders by clicking File --> Import. First, a structure file has to be loaded. Within the structure menu, several trajectories can be imported via Remote trajectory import. The content of the simulation can be filtered (filter selection) and within the trajectory menu, the simulation can be further adjusted (e.g. centering, superposing, removing PBC, step sizes or play options).

If you want to load a trajectory containing already all atomistic informations (as .gro trajectories), select File --> as trajectory before loading the file.

File description

Configuration file

The configuration file can be (optionally) used to locate the directories that will be accessible though the web server and it defines access restrictions as user and password settings. It is written in Python and a sample .cfg file is provided.

.ngl file

The .ngl file is written in JavaScript and contains commands to execute a session. A sample .ngl file shows additional options like functions and basic loading commands. This is recommended if simulation sessions should be shared. The location of this file should be within an accessible directory and can then be loaded via the URL (e.g. http://localhost/mdsrv.html?load=file://MDsrv/example.ngl, where MDsrv is the directory defined within the .cfg file)

.html file

The .html file loads the NGL viewer including the MDsrv configurations. An example file can be inspected.
